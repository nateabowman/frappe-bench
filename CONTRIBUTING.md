# Contributing

Thank you for your interest in contributing.

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Ensure no sensitive data (passwords, API keys, real domains) is committed
5. Submit a pull request

## Development Setup

```bash
bench start          # Start development server
bench --site all migrate   # Run migrations
bench build          # Build assets
```

## Code Style

- Follow existing code style in the project
- Use meaningful commit messages
- Keep documentation updated

## Documentation

Documentation lives in the `docs/` directory. Update relevant docs when adding features or changing behavior.

## Questions?

Open an issue for questions or discussions.
